" A programming language is a set of rules that provides a way of telling a computer what operations to perform "

Besides... the source code is just plain text, there is nothing magical about it. 


if (users.length > 1)
{
	console.log("Hello to all users");
}
else if (users.length == 1)
{
	console.log("Hello to user " + users[0].name);
}
else
{
	console.log("It's lonely here, there are no users!");
}

Source code needs to be converted into the machine language in order to be executed. There are two main ways to convert the source code: by compiling it and by interpreting it.

When we deal with:

  • Compiled languages, the source code is converted by the compiler into the machine language, generating an executable file.
  • Interpreted languages, the source code is given to an interpreter which converts and runs it "on the fly".

An example of interpreters: the Browsers

The browsers interpret the languages of the web, such as:

  • JavaScript
  • HTML
  • CSS

HTML (HyperText Markup Language) is not really a programming language

generally, a programming language allows you to describe some sort of process of doing something, whereas HTML is just a way of adding context and structure to text,
but it still gives us the idea of what an interpreter does...

When the browser receives some HTML code like this:


<h1>Here's a title</h1>
<p>
	Here's a paragraph.<br>
	It contains some <strong>bold text</strong> and some <i>italic text</i><br>
	It also contains a <a href="http://www.google.com" target="_blank">link to Google</a>.
</p>

it outputs the following:

Here's a title

Here's a paragraph.
It contains some bold text and some italic text
It also contains a link to Google.
